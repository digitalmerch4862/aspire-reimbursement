# Employee Database — Payee-List CRUD + Mass Merge Upload

**Date:** 2026-06-26
**Area:** `App.tsx` — System Settings → Employee Database section
**Goal:** Replace the single raw-textarea employee editor with a NAB-style "payee list":
per-employee Edit/Delete via modal (single update), plus a file upload that smart-merges
the whole roster (mass update) and accepts Excel as well as CSV.

---

## Problem / Motivation

Today the Employee Database section is one big `<textarea>` plus a CSV/TXT uploader that
**replaces** the whole active list. Two gaps:

1. **No single-record editing.** When one employee occasionally changes their bank
   details, the only path is hand-editing a 600+ line textarea — error-prone.
2. **Upload can't read the real file.** The roster arrives as `.xlsx`
   (`Updated Details (26.06.26).xlsx`); the uploader only accepts `.csv/.txt`, so it
   silently does nothing useful with the actual file.

A latent bug compounds this: `serializeEmployeeData` writes a **comma**-delimited header
while `saveEmployeesToSupabase` splits each line on **tab**. Any code path that serializes
then syncs (e.g. "Keep Active") pushes null `first_name/surname/...` rows to Supabase.

## Goals

- **Single update:** searchable, paginated list of employee cards; each has a kebab menu →
  Edit (modal form) / Delete (confirm). An "Add Employee" button opens the same modal in
  add mode.
- **Mass update:** one Upload File button accepting `.xlsx/.csv/.txt`, smart-merging by
  Account number (update matches, add new, current-missing-from-file → approval queue).
- Fix the serialize/Supabase delimiter mismatch as part of this work.

## Non-Goals (YAGNI)

- No new stored fields. Employee model stays `{id, firstName, surname, fullName, bsb, account}`
  (NAB's Nickname / Reference / Remitter / Account-name are **not** persisted).
- No BSB → bank-name lookup, no "Verify account details" API.
- No Supabase schema change.
- No "replace all" upload mode (smart-merge only).

---

## Data Model (unchanged)

```ts
interface Employee {
  id: string;        // stable internal key; generated by parseEmployeeData / add flow
  firstName: string;
  surname: string;
  fullName: string;  // `${firstName} ${surname}`
  bsb: string;
  account: string;
}
```

- **Edit identity = `id`** (so editing the Account number itself works).
- **Mass-merge identity = `account`** (uploaded files carry no internal id).
- Account-name shown in UI is derived, not stored: `\`${surname}, ${firstName}\`.toUpperCase()`.

## Components / Changes (all in `App.tsx` unless noted)

### 1. Shared persistence + mutation helpers

- `persistEmployeeList(next: Employee[]): Promise<void>`
  Central writer: `setEmployeeList(next)` → `setEmployeeRawText(serializeEmployeeData(next))`
  → `localStorage.setItem('aspire_employee_list', serialized)` → if `hasSupabaseEnv`
  `await saveEmployeesToSupabase(serialized)`. Replaces the duplicated persist blocks in
  `handleSaveEmployeeList` and `handleKeepPendingEmployee`.
- `upsertEmployeeById(rec: Employee)` → next list with rec replacing same-`id`, else appended.
- `removeEmployeeById(id: string)` → next list without that id.
- `mergeEmployeesByAccount(current: Employee[], incoming: Employee[])`
  → `{ merged: Employee[]; pendingDeactivation: Employee[] }`.
  - Key by trimmed `account`.
  - For each incoming row: account matches a current employee → update its
    firstName/surname/bsb/fullName (keep current `id`); else add as new (fresh id).
  - Current employees whose account is **absent** from the incoming file →
    `pendingDeactivation` (NOT deleted; surfaced in the existing approval queue).

### 2. `serializeEmployeeData` — delimiter fix

Emit **tab**-delimited with a `Concatenate` column so it round-trips through both
`parseEmployeeData` (auto-detects delimiter) and `saveEmployeesToSupabase` (tab split):

```
First Names<TAB>Surname<TAB>Concatenate<TAB>BSB<TAB>Account
First<TAB>Surname<TAB>Surname, First<TAB>bsb<TAB>account
```

### 3. xlsx ingestion in `handleCsvFileChange`

- `import readXlsxFile from 'read-excel-file'` (already a dependency).
- Branch on file name/extension:
  - `.xlsx` → `const rows = await readXlsxFile(file)` → map each row to a tab-joined line,
    prepend the standard header → feed that string to the existing parse/merge path.
  - else → `await file.text()` as today.
- Column mapping for xlsx mirrors `parseEmployeeData` header detection (First Names,
  Surname, BSB, Account); when the sheet has a header row, match by name, else fall back
  to positional columns.

### 4. Upload flow → smart merge + preview

- Parse incoming (csv/txt/xlsx) → `incoming: Employee[]`.
- `mergeEmployeesByAccount(employeeList, incoming)`.
- Show preview message: `"X updated, Y new, Z moved to approval-to-deactivate"`.
- Apply: `persistEmployeeList(merged)` + `persistPendingDeactivationEmployees(pendingDeactivation)`.
- Approval-to-deactivate queue UI (Keep Active / Approve Deactivate) is retained as-is.

### 5. List UI (payee-list)

State (view-only, not persisted): `employeeSearch: string`, `employeePage: number`,
`employeePageSize: number` (default 10; options 10/25/50).

- Header: **Add Employee** (primary) · **Upload File** · **Reset Defaults**.
- **Search** input filters `employeeList` by case-insensitive substring over
  displayName / account / bsb.
- Derived `filtered` → `paged` slice; show `"{start}–{end} of {filtered.length}"`,
  page-size selector, prev/next.
- Card per employee: bold display name; `Account name` = `SURNAME, FIRSTNAME`; a Type chip
  (label `Active`); `BSB`; `Account number`; kebab ⋮ → **Edit** / **Delete**.
- Collapsible **Advanced (raw edit)**: the existing textarea + Save Changes button, kept as
  an escape hatch (Save Changes routes through `persistEmployeeList(parseEmployeeData(text))`).

### 6. Add / Edit modal

State: `employeeModal: { mode: 'add' | 'edit'; draft: Employee; errors: Record<string,string> } | null`.

- Fields: First Name, Surname, BSB, Account number. Live preview "Account name:
  SURNAME, FIRSTNAME".
- Validation (block Save, inline messages):
  - firstName / surname: non-empty after trim.
  - bsb: exactly 6 digits (`/^\d{6}$/`).
  - account: 6–10 digits (`/^\d{6,10}$/`).
- Save:
  - add mode → build Employee with fresh id → `upsertEmployeeById` → `persistEmployeeList`.
  - edit mode → keep existing id, recompute fullName → `upsertEmployeeById` → `persistEmployeeList`.
- Cancel discards draft.

### 7. Delete

- Kebab → Delete → confirm dialog (name + account) → `removeEmployeeById` →
  `persistEmployeeList`. Direct removal (admin-initiated), independent of the approval queue.

## Error Handling

- Upload: unreadable file / zero valid rows → existing `csvImportMessage` error text;
  reset the file input.
- xlsx read throw → caught, user-facing "Could not read Excel file" message.
- Supabase sync failure inside `persistEmployeeList` → keep local state, surface
  `showWarningToast` (same pattern as current `handleSaveEmployeeList`).
- Modal validation failures never call persistence.
